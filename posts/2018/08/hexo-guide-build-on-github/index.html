搭建Hexo

1.用你的软件包管理器装好node.js,npm和git
2.用npm安装hexo和hexo-deployer-git
3.在github上用你的账号注册项目”{用户名}.github.io”
4.准备一个用来装博客的文件夹,然后执行

新版的hexo在这一步会自动执行npm install来补全依赖
如果没有的话手动执行一遍就可以了

5.然后生成博客的文档 

hexo g

6.(非必要)在新的终端执行 

hexo s

访问提示出来的链接
如果并不能进hexo,用 

hexo s -p 端口号

换一个新的端口试试

连接到github

7.用 

git config –global user.name “用户名” 
git config –global user.email “邮箱”

登陆至github

8.生成ssh密钥: 

ssh-keygen -t rsa -C “邮箱”

然后基本上就三个回车就可以了 不强制要求加密码
然后添加密钥到ssh-agent 

eval “$(ssh-agent -s)” && ssh-add ~/.ssh/id_rsa

然后打开~/.ssh/id_rsa.pub 在github账户设置里用这个文件的内容创建新的ssh key

9.(非必要)测试ssh: 

ssh -T git@github.com

如果提示Hi “github用户名” 就说明成功，否则删除ssh key再重新生成 

ssh-add -D && rm -r ~/.ssh

10.在博客的文件夹里找到_config.yml，在最后找到deploy 

type: git  
repo: git@github.com:{用户名}/用户名.github.io.git  
branch: master  
message: {提示信息} (非必要)

然后！一定要记得打空格！ 

type:[ ]git  
repo:[ ]git@github.com:{用户名}/用户名.github.io.git  
branch:[ ]master  
message:[ ]{提示信息} (非必要)

这也是YAML的基本注意事项。

11.部署博客到github: 

hexo g && hexo d

稍等大概1分钟，然后去博客看一下(｡･∀･)ﾉﾞ

添加文章

用如下指令创建新的文章: 

hexo new post 标题

或者对于文章(post)，可以直接用

hexo new 标题

创建新文件。
然后去source文件夹下找新建的.md文档
就可以愉快的开始写文章啦(｡･∀･)ﾉﾞ
写完之后直接 hexo g && hexo d就可以推送到github上啦
或者 

hexo d -g

也是可以的啦(｡･∀･)ﾉﾞ
